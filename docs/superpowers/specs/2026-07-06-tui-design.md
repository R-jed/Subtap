# Subtap TUI 设计方案

> 最终版 · 2026-07-06

---

## 一、设计原则

- **直接移植 Mole 的 TUI 模式到 Python**，不依赖 Rich 做菜单和交互
- **扁平风格**，靠颜色区分层级，不使用框线
- **产品语言**，面向普通用户，不用工程术语
- **vim 键位 + 方向键**，vim 键位静默支持，不在 UI 提示
- **引导式流程**，首次使用自动进入初始化配置
- **智能默认值**，用户可以直接按 Enter 往下走，不需要理解每个选项

---

## 二、启动方式

```bash
# 交互式菜单
$ subtap

# 快捷转录
$ subtap recording.mp3
$ subtap recording.mp3 --lang zh --format srt

# 转录历史
$ subtap history

# 批量转录
$ subtap batch ./audio-files/
```

无参数进入 TUI 菜单，有参数走快捷路径。

## 三、初始化引导（首次使用）

检查 `~/.subtap/config.yaml` 是否存在 + 是否有有效的 API key 或已下载模型，不满足则自动进入。

### 欢迎

```
  欢迎使用 Subtap

  首次使用，需要完成基础配置

  Enter 开始配置
```

### 步骤 1/2 · 配置服务

```
  配置 · 步骤 1/2

  接口地址：https://api.example.com/v1
  密钥：sk-********************************

  Tab 切换  Enter 确认
```

默认填入常见服务地址，用户只需粘贴密钥。本地模式和模型选择收进"设置"，不在首次引导中出现。

```

  ➤ 1. 中文    自动检测中文方言

```

## 五、新建转录

### 文件选择

```
  新建转录

  Enter 选择音频或视频文件

  支持格式：mp3, wav, m4a, mp4, mkv, avi

  Enter 选择文件  Esc 返回
```

### 确认页

```
  新建转录 · 确认

  文件：会议录音.mp3
  时长：32:15
  大小：45.2 MB

  语言：中文（自动检测）
  格式：SRT

  Enter 开始转录  S 更多设置  Esc 返回
```

只保留语言和格式两个最常用选项，其他用智能默认值。想改的用户按 `S` 进详细设置。

### 转录进度

```
  正在转录 · 会议录音.mp3

  ⠙ 听录音中...  ████████░░░░░░░░  52%
  · 整理文字
  · 保存文件

  预计剩余：3:20

  Q 取消
```

内部 7 个阶段（分块、识别、清洗、分句、学习、翻译、导出）折叠为 3 个用户能理解的阶段。细节日志可选展开。

### 转录完成

```
  转录完成 · 会议录音.mp3

  耗时：5:32
  识别：12,847 字  186 条字幕

  输出文件：
    ~/Documents/会议录音.srt
    ~/Documents/会议录音.txt

  ➤ 1. 打开文件
    2. 打开文件夹
    3. 新建转录

  ↑↓ 导航  Enter 确认  Esc 返回主菜单
```

路径显示用 `~` 缩写。`1` 用系统默认程序打开文件，`2` 在 Finder 中显示。用数字菜单替代单字母快捷键，和主菜单风格统一。

## 十、键盘交互

| 按键 | 作用 |
|------|------|
| `↑` `k` | 上移 |
| `↓` `j` | 下移 |
| `←` `h` | 左移（值切换） |
| `→` `l` | 右移（值切换） |
| `Enter` | 确认/选择 |
| `Esc` | 返回上级 |
| `Q` | 退出（主菜单时） |
| `1`-`9` | 快速选择菜单项 |
| `S` | 更多设置（确认页） |
| `D` | 删除（历史列表） |
| `gg` | 跳到列表顶部 |
| `G` | 跳到列表底部 |
| `Tab` | 切换输入框 |

ESC 序列兼容 CSI（`ESC[A`）和 SS3（`ESC O A`），确保 iTerm2、Terminal.app、tmux、SSH 等环境方向键可用。

防抖机制：`drain_pending_input` 10ms 超时排空缓冲区，防止快速输入导致菜单跳跃。

vim 键位（j/k/h/l/gg/G）静默支持，不在状态栏提示。普通用户只看到 `↑↓ 导航`。

## 十一、颜色体系

从 Mole 移植 8 色语义映射：

| 语义 | ANSI 色 | 用途 |
|------|---------|------|
| 品牌 | 绿色 `\033[0;32m` | Logo、完成 `✓` |
| 选中 | 青色 `\033[0;36m` | 当前选中项 `➤` |
| 标题 | 紫色 `\033[0;35m` | 页面标题 |
| 提示 | 灰色 `\033[0;90m` | 状态栏、副标题 |
| 信息 | 蓝色 `\033[1;34m` | 文件路径 |
| 警告 | 黄色 `\033[0;33m` | 大文件、部分失败 |
| 错误 | 红色 `\033[0;31m` | 失败 `✗` |
| 等待 | 灰色 `\033[0;90m` | 待处理阶段 `·` |

支持 `NO_COLOR` 环境变量，设置后禁用所有颜色。

## 十二、文件结构

```
src/subtap/ui/
├── tui.py            # TUI 主入口，页面路由
├── menu.py           # 菜单系统（渲染、选中、滚动）
├── keyboard.py       # 键盘输入（CSI+SS3 解析、防抖、vim 键位）
├── theme.py          # 颜色语义映射、NO_COLOR 支持
├── input.py          # 文本输入框（密钥等）
├── views/
│   ├── setup.py      # 初始化引导（2 步配置）
│   ├── home.py       # 主菜单
│   ├── new_task.py   # 新建转录（选择、确认、进度、完成）
│   ├── history.py    # 转录历史（列表、详情）
│   ├── batch.py      # 批量转录
│   └── settings.py   # 设置（5 个子页面）
├── state.py          # PipelineState（已有，复用）
├── runner.py         # RichRunner（已有，保留用于非交互 Pipeline 输出）
├── dashboard.py      # Textual 仪表板（已有，保留可选 --tui 模式）
├── event_bridge.py   # 事件桥接（已有，复用）
└── observer.py       # 观察者（已有，保留）
```

## 十三、技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 菜单渲染 | ANSI 原生（移植 Mole） | 轻量、零新增依赖、Mole 已验证 |
| 键盘输入 | 移植 Mole `read_key()` | CSI+SS3 兼容、vim 键位、防抖 |
| Pipeline 输出 | Rich（已有） | 非交互场景，已有代码复用 |
| 全屏监控 | Textual（已有，可选） | `--tui` 模式保留 |
| 主题 | ANSI 8 色语义映射 | 移植 Mole 颜色体系 |

菜单系统不依赖 Rich，直接移植 Mole 的 ANSI 模式。Rich 仅保留给现有 RichRunner 的非交互式 Pipeline 输出。

## 十四、实现阶段

| 阶段 | 内容 | 验证 |
|------|------|------|
| P0 | 修复阻塞问题（textual 依赖声明、command_deck 引用清理） | `subtap --tui` 不报 ImportError |
| P1 | 键盘输入层（`keyboard.py`，移植 Mole `read_key` + 防抖） | 方向键 + vim 键位在各终端可用 |
| P2 | 菜单框架（`menu.py` + `theme.py`） | 主菜单可交互导航 |
| P3 | 初始化引导（`setup.py`） | 首次 `subtap` 自动进入配置流程 |
| P4 | 新建转录流程（文件选择 → 确认 → 进度 → 完成） | 端到端转录可走通 |
| P5 | 转录历史（列表、详情、删除确认、重新保存） | 历史记录可查看和操作 |
| P6 | 设置系统（5 个子页面） | 各设置项可修改并持久化 |
| P7 | 批量转录 | 多文件队列可执行 |
